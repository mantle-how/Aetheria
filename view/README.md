# 視圖層說明

`view/` 是展示層，負責把模擬資料畫出來，不應承載領域決策邏輯。

目前使用 `tkinter` 提供兩個同步視覺化：

- 上帝視角的 2D 俯視圖
- 統一的代理人儀表板（需求線圖 + 食物剩餘）

## 檔案

- `demo.py`：可直接執行的示範入口。
- `topdown.py`：`TopDownVisualizer`，負責繪製、播放控制與多視窗同步更新。

## demo.py

`view/demo.py` 的流程是：

1. 建立 `SimulationConfig`
2. 建立示範模擬
3. 建立 `TopDownVisualizer`
4. 把實體、步進與狀態文字 callback 傳給 `show_live()`
5. 同時開啟俯視圖主視窗與代理人儀表板視窗

常用啟動方式：

```bash
python view/demo.py
```

## topdown.py

`TopDownVisualizer` 支援：

- `show(entities)`：顯示靜態畫面
- `show_live(...)`：依時間間隔持續更新畫面，並同步更新代理人儀表板

主要責任包括：

- 計算世界邊界與縮放比例
- 將世界座標轉成畫布座標
- 依實體類別套用顏色
- 在俯視圖中顯示 agent 的名字與目前動作
- 繪製圖例與狀態列
- 在儀表板中為每位 agent 顯示：
  需求折線圖（飢餓、精力、心情）
  食物剩餘數量與長條
  目前動作與即時數值
- 儀表板卡片會依視窗寬度自動換列排列

## 鍵盤操作

- `Space`：播放或暫停
- `+`：放大
- `-`：縮小
- `Esc`：關閉視窗

補充：

- 儀表板有垂直捲軸，可在 agent 較多時往下查看
- 關閉儀表板後，主視窗仍可繼續播放

## 分層邊界

若未來更換 UI 技術，這個資料夾應是主要替換點。領域邏輯應留在 `domain/`，不要回寫到視覺層。
